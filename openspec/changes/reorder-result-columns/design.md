## Context

Currently, the output CSV file generated by the application does not have its columns ordered in a way that matches the expected `sample.csv`. This causes downstream friction. The goal is to enforce a specific column order where all fields are aligned with `sample.csv`, and specifically, the "Trạng thái" (Status) and "Độ tự tin" (Confidence) columns are appended as the final two columns.

## Goals / Non-Goals

**Goals:**
- Ensure the exported CSV file has the exact same column order as `sample.csv` (for matching columns).
- Guarantee that "Trạng thái" and "Độ tự tin" are the last two columns in the file, regardless of their position in the intermediate dataframes.

**Non-Goals:**
- Altering the logic of how "Trạng thái" or "Độ tự tin" are calculated.
- Renaming existing columns in the output other than necessary formatting.

## Decisions

- We will modify the data export function (likely in Python backend using pandas) to explicitly select and reorder the DataFrame columns before saving to CSV/Excel.
- The reference order will be statically defined or dynamically extracted from a `sample.csv` header if provided, followed by explicitly appending the two metadata columns at the end.

## Risks / Trade-offs

- **Risk: Missing columns.** If the generated data is missing a column expected by `sample.csv`, pandas might throw a KeyError during reordering. -> **Mitigation:** Use safe reindex or `df.reindex(columns=...)` which fills missing columns with NaNs, or explicitly filter to only reorder existing columns.
